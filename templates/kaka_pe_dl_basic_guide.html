1. 빠른 시작 (5단계)

  1. 쿠키 주입 — 설정 > 일반의 카카오 쿠키에 로그인 쿠키(JSON)를 붙여넣습니다. (→ 2번)
  2. 다운로드 경로 지정 — 다운로드 경로에서 저장할 폴더를 선택합니다.
  3. 작품 등록 — 웹툰은 체크할 웹툰, 웹소설은 체크할 소설에 한 줄에 하나씩 입력 (제목 / URL / content/숫자 / 숫자 ID 모두 가능).
  4. 설정 저장 → 쿠키 검증 — 상단 설정 저장 후 쿠키 검증으로 로그인 상태를 확인합니다.
  5. 실행 — 즉시 실행으로 바로 받거나, 설정 > 자동 탭에서 스케줄러를 등록합니다. 진행 상태는 진행 상황 탭에서 확인합니다.
작품 입력칸의 ID → 제목 풀이 버튼으로 등록한 토큰이 어떤 작품(웹툰/소설)인지 미리 확인하고, 정렬 + 중복 제거로 목록을 정리할 수 있습니다.

2. 카카오 쿠키 준비하기

이 플러그인은 사용자의 카카오 로그인 세션(쿠키)으로 카카오페이지에 접근합니다. 비밀번호는 저장하지 않습니다.

  1. 크롬에서 https://page.kakao.com 에 로그인합니다.
  2. 크롬 확장 Cookie-Editor를 설치합니다.
  3. 카카오페이지 탭에서 Cookie-Editor를 열고 Export → JSON 으로 복사합니다.
  4. 복사한 JSON을 설정 > 일반 > 카카오 쿠키 칸에 붙여넣고 설정 저장.
  5. 쿠키 검증 버튼으로 유효 여부를 확인합니다.
필요 쿠키: _kau, _kpwtkn, _T_ANO, _karmt, _kahai, _kawlt, _kpdid. 보통 약 30일 유지되며, 만료되면 다시 export해서 주입하세요. 웹훅을 설정해두면 쿠키 만료 시 1회 알림을 받습니다. (→ 9번)

알림 탭

  • 웹툰 다운로드 완료 알림 — 자동 실행 후 받은 웹툰이 있을 때 요약 발송.

자동 탭

  • 시작시 자동실행 — On이면 SJVA 시작 시 스케줄러 자동 등록.

4. 대여권/티켓과 회차 다운 규칙

스케줄러(또는 즉시 실행)는 등록한 작품의 받지 않은 회차를 받습니다. 회차 상태별 처리는 다음과 같습니다.

구분처리
무료 / 소장대여권 없이 바로 받습니다.
기다무기다무 대여권 사용 On이고 충전돼 있으면 기다무 대여권으로 받습니다(1회 최대 장수 한도).
잠금(대여 필요)일반 대여권 사용 On이고 보유 대여권이 있으면 받습니다. 둘 다 Off면 스킵.
대여권을 쓰는 회차는 카카오에서 실제로 대여권이 차감됩니다. 의도치 않은 차감을 막으려면 옵션을 끄고 무료/소장만 받으세요. 이 플러그인은 캐시/현금으로 대여권을 구매하지 않습니다.

5. 수동 다운로드

수동 다운로드 탭은 작품 하나를 즉석에서 골라 받을 때 씁니다.

  1. 작품/회차 URL(또는 ID)을 입력하고 분석.
  2. 회차 목록에서 받을 회차를 체크합니다(받기 가능한 회차만 선택 가능 — 이미 받은 회차 제외).
  3. 선택 다운로드로 직렬 다운로드. 취소로 중단.
수동 다운로드는 직접 누르는 동작이라 자동 다운로드와 독립적입니다. 단, 압축 등 다른 작업이 돌고 있으면 충돌 방지를 위해 끝난 뒤 다시 시도하라는 안내가 뜹니다.

6. 저장 구조와 중복 처리

폴더 구조
다운로드경로/
├─ webtoon/
│  └─ 작품제목/
│     ├─ info.xml / cover.jpg
│     ├─ 0001_회차제목/        (압축 Off)
│     │   ├─ 001.jpg ...
│     └─ 0002_회차제목.zip     (압축 On — 폴더 대신 zip)
└─ novel/
   └─ 작품제목/
      └─ 0001_회차제목.txt     (소설은 회차당 txt 1개, 압축 안 함)
이미 받은 회차는 어떻게 판단하나?
  • 기본 판단 기준은 DB의 다운로드 이력(회차 단위 completed)입니다. 이력에 완료로 있으면 다시 받지 않습니다.
  • 웹툰 + 압축(ZIP) 사용 시: DB 이력에 없더라도 다운로드 폴더에 해당 웹툰 회차 zip이 이미 있으면 다시 받지 않고 이력에만 인식(등록)합니다. → 같은 회차를 헛다운로드하거나 대여권을 다시 쓰는 일을 막습니다.
  • zip은 정상 완료된 웹툰 회차에만 만들어지므로 “zip 있음 = 온전히 받은 회차”로 안전하게 간주합니다. 받다 만 폴더(zip 없음)는 다시 받습니다.
  • 압축 처리 중 이미 같은 이름의 zip이 있으면 덮어쓰지 않고 기존 zip을 유지합니다.
처음 도입해서 “예전에 받아둔 웹툰 zip”을 이력에 인식시키고 싶을 때, 압축을 켠 상태로 자동/수동 실행을 한 번 돌리면 해당 작품의 기존 zip들이 재다운로드 없이 이력에 등록됩니다.
압축을 끄고 폴더로 보관하거나, 소설(.txt)인 경우엔 이 “디스크 인식”이 동작하지 않습니다(완전 다운 여부를 zip 으로만 보장하므로). 웹툰 중복을 디스크로 방지하려면 압축을 켜는 것을 권장합니다.

8. 진행 상황 · 다운로드 이력

  • 진행 상황 — 자동/수동 다운로드의 현재 상태와 작품·회차·페이지 진행률을 실시간으로 표시.
  • 다운로드 이력 — 받은 회차 목록. 상태 필터 · 제목 검색 · 웹툰/소설 종류 필터를 지원하고 선택 삭제가 가능합니다.

9. 알림(웹훅) · 프록시

웹훅 알림
  • Discord 웹훅 URL 예: https://discord.com/api/webhooks/{ID}/{TOKEN} (Slack 웹훅도 자동 인식).
  • 쿠키 만료 / 웹툰 완료 / 소설 완료 알림을 각각 켤 수 있습니다. (설정 > 알림)
프록시 (선택)
  • 프록시 사용 On + URL 값이 있을 때만 적용. 예) http://user:pass@host:8080 · socks5h://host:1080.
  • On이면 API 호출·이미지 CDN 다운로드가 같은 프록시를 사용합니다(토큰 IP 검증 회피).
  • 트래픽 부담이 커지므로 불안정하면 끄세요.

10. 문제 해결 (FAQ)

받아지는 게 0개예요 / “쿠키 만료” 가 떠요

쿠키 만료 가능성이 큽니다. 카카오페이지에 다시 로그인 → Cookie-Editor로 재-export → 쿠키 칸에 붙여넣고 설정 저장 → 쿠키 검증으로 확인하세요.

잠금 회차가 안 받아져요

기다무 회차는 기다무 대여권 사용을, 보유 대여권이 있는 회차는 일반(보유) 대여권 사용을 켜세요. 대여권이 없으면 잠금 회차는 스킵됩니다(대여권을 구매하지 않음).

같은 웹툰 회차를 자꾸 다시 받아요

압축(ZIP)을 켜두면 디스크의 기존 zip을 인식해 재다운로드와 대여권 재차감을 피합니다. (→ 6번) 폴더 보관(압축 Off)이나 소설은 디스크 인식 대상이 아닙니다.

소설이 안 받아져요 / 웹툰만 받아져요

소설은 체크할 소설 칸에 따로 등록해야 합니다. ID → 제목 풀이로 토큰이 웹툰/소설 중 무엇으로 인식되는지 확인하세요.

스케줄이 안 도는 것 같아요

설정 > 자동에서 스케줄러 등록·실행 상태를 확인하고 로그 탭에서 실행 로그를 확인하세요. 즉시 실행으로 먼저 동작을 점검할 수 있습니다.

※ 이 플러그인은 본인이 정당하게 접근 권한을 가진 콘텐츠를 개인 보관 목적으로 내려받는 용도입니다. 카카오 이용약관과 저작권법을 준수해 사용하세요.
